# Security reporting

Use this repository's private vulnerability reporting flow for a suspected
security problem. Do not open a public issue containing exploit details or
credentials.

Do not attach an Eido save, Card text, prompts, generated outputs, images or
other assets, API keys, signing material, access tokens, private filesystem
paths, or unredacted diagnostic bundles. A maintainer will ask for the smallest
safe reproduction if more evidence is needed.

Never disable Gatekeeper, SmartScreen, signature verification, antivirus, or
another operating-system protection to work around an Eido problem.
