# Support

Use a public issue for non-sensitive release, installation, update, or
documentation problems. Include the operating system, Eido version, artifact
name, and published checksum when relevant.

Do not include Card or Board content, prompts, assets, API keys, credentials,
private paths, or an Eido save. Use private vulnerability reporting for a
security-sensitive report.

This repository is public and downloads are not access-controlled. A link being
shared beyond the Friends Alpha cohort is expected behavior, not a security
incident.
